# Fase 3: Admin CRUD Pesantren + Profile Page

## 📋 Deskripsi
Melengkapi fitur admin dengan CRUD pesantren dan menambahkan halaman profile untuk semua user yang terautentikasi.

## 🎯 Scope

### Komponen yang Diimplementasi

| Komponen | Path | Role | Fitur |
|----------|------|------|-------|
| **AdminPesantrenManagement** | component | Superadmin | CRUD pesantren, delete confirmation, validation |
| **AdminDashboard** (update) | `/dashboard` | Superadmin | Tab: Pendaftaran + Pesantren management |
| **ProfileView** | `/profile` | All authenticated | Update username/email, change password |
| **AppNavbar** (update) | component | All | Show username + profile link |

### Fitur Detail

#### 1. Admin Pesantren Management
- Tabel pesantren dengan kolom: nama, lokasi, kurikulum, santri, biaya
- Tambah pesantren baru via modal form
- Edit pesantren dengan pre-filled data
- Hapus pesantren dengan konfirmasi modal
- Validasi: nama min 3 chars, province/kota required, email format, phone format, year range, numeric min 0
- Fasilitas input comma-separated

#### 2. Admin Dashboard (Update)
- Tab navigation: "Manajemen Pendaftaran" + "Manajemen Pesantren"
- Pendaftaran tab: inline status update, detail modal, export Excel
- Pesantren tab: full CRUD dengan AdminPesantrenManagement component
- Stats cards tetap di atas

#### 3. Profile Page
- Section 1: Update informasi akun (username, email)
  - Validasi: username 3-30 chars alphanumeric, email format
  - Show role (disabled field)
  - Submit disabled sampai ada perubahan (meta.dirty)
- Section 2: Ubah password
  - Current password, new password, confirm password
  - Validasi: min 6 chars, password match
  - Auto-logout setelah berhasil ganti password

#### 4. Navbar Update
- Tampil username authenticated user dengan icon 👤
- Link ke /profile
- Tetap ada Dashboard link dan tombol Keluar

## 🛠 Tech Stack
- **VeeValidate + Yup** - Form validation (consistent dengan Fase sebelumnya)
- **Bootstrap 5** - Tabs, modals, tables, forms, alerts
- **Vue 3** - Composition API, defineEmits for component communication

## ✅ Acceptance Criteria
- [x] Admin bisa CRUD pesantren dari dashboard
- [x] Delete confirmation modal dengan error handling
- [x] Admin dashboard punya tab navigation
- [x] User bisa update username dan email
- [x] User bisa ganti password dengan validasi
- [x] Auto-logout setelah ganti password
- [x] Navbar menampilkan username + link profile
- [x] Build berhasil tanpa error
