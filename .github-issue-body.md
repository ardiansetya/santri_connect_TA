# Fase 2: Implementasi Halaman Authenticated Frontend

## 📋 Deskripsi
Melanjutkan implementasi frontend dengan menambahkan semua halaman yang memerlukan autentikasi. Dashboard role-based untuk 3 role user: pendaftar, pemilik, dan superadmin.

## 🎯 Scope

### Komponen yang Diimplementasi

| Komponen | Path | Role | Fitur |
|----------|------|------|-------|
| **DashboardView** | `/dashboard` | All | Role-based routing, show correct dashboard |
| **PendaftarDashboard** | component | Pendaftar | Stats, riwayat pendaftaran, quick actions |
| **PemilikDashboard** | component | Pemilik | CRUD pesantren sendiri, modal form |
| **AdminDashboard** | component | Superadmin | Stats, manajemen pendaftaran, export Excel |
| **PendaftaranFormView** | `/pesantren/:id/daftar` | Pendaftar | Form pendaftaran + upload dokumen |

### Fitur Detail

#### 1. PendaftarDashboard
- Stats cards: Pendaftaran Aktif, Diterima, Total
- Riwayat pendaftaran dengan status badge
- Quick actions: Cari Pesantren, Rekomendasi, Bandingkan, Cek Status
- Empty state dengan CTA

#### 2. PemilikDashboard
- Stats: Jumlah Pesantren, Total Santri
- List pesantren milik sendiri
- Tambah/Edit pesantren via modal form
- Form lengkap: nama, lokasi, kurikulum, biaya, kontak, fasilitas
- Validasi max 5 pesantren

#### 3. AdminDashboard
- Stats: Total Pesantren, Pendaftaran, Pending, Diproses
- Tabel pendaftaran dengan inline status update (dropdown)
- Detail pendaftaran dalam modal
- Export data pendaftaran ke Excel
- Download otomatis dengan timestamp

#### 4. PendaftaranFormView
- 3 section: Data Pribadi, Data Orang Tua, Dokumen
- Upload: Foto KTP, Pas Foto, Kartu Keluarga
- Success state dengan nomor pendaftaran + link cek status
- Auto-load pesantren info dari URL params

### Perubahan Infrastruktur

| File | Perubahan |
|------|-----------|
| `router/index.js` | Tambah route pendaftaran, auth guard redirect ke login |
| `stores/auth.js` | Tambah action `updateProfile()` |
| `PesantrenDetailView.vue` | Tambah tombol "Daftar ke Pesantren Ini" |

## 🛠 Tech Stack
- **Vue 3** - Composition API (`<script setup>`)
- **Bootstrap 5** - Cards, Forms, Tables, Modals, Badges, Alerts, Grid
- **Pinia** - State management
- **Vue Router** - Navigation guards dengan role-based redirect

## ✅ Acceptance Criteria
- [x] Dashboard menampilkan komponen berbeda sesuai role
- [x] Auth guard redirect ke login jika belum auth
- [x] Pemilik bisa CRUD pesantren sendiri
- [x] Admin bisa update status pendaftaran inline
- [x] Admin bisa export data ke Excel
- [x] Form pendaftaran dengan file upload berfungsi
- [x] Build berhasil tanpa error
