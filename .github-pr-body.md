## Summary

- Implementasikan **5 halaman publik** frontend yang sebelumnya masih placeholder
- Ganti semua vanilla CSS styling menjadi **Bootstrap 5** utility classes
- Tambah **Compare Store** (Pinia) untuk mengelola daftar bandingkan
- Lengkapi **service layer** dengan admin, pemilik, wilayah, dan pendaftaran endpoints
- Tambah **Swagger/OpenAPI** documentation di backend

## 🎨 Halaman yang Diimplementasi

### 1. Pesantren List (`/pesantren`)
- Bootstrap grid card layout (`row-cols-1 row-cols-md-2 row-cols-lg-3`)
- Filter provinsi & kurikulum
- Sorting: nama, tahun berdiri, jumlah santri, biaya
- Pagination
- Tombol "Bandingkan" terintegrasi Compare Store

### 2. Pesantren Detail (`/pesantren/:id`)
- Breadcrumb navigation
- 2-column layout: detail utama + sticky sidebar
- Sections: Informasi Umum, Biaya, Fasilitas, Kontak
- Info pendaftaran dari API endpoint

### 3. Rekomendasi (`/rekomendasi`)
- Form: budget, provinsi, fasilitas (checkboxes)
- Score breakdown dengan Bootstrap progress bars
- Info card cara kerja scoring (40/30/30)

### 4. Compare (`/compare`)
- Side-by-side comparison table (Bootstrap `table-bordered`)
- Auto-load dari Compare Store
- Hapus individual / hapus semua
- **Fix bug**: import `pakistan` → `pesantren`, `PesantrenData` → `pesantrenData`

### 5. Track Status (`/track`)
- Form input nomor pendaftaran
- Timeline visual (Pending → Diproses → Selesai)
- Detail cards, status badges, catatan admin

## 📦 Infrastruktur Baru

| File | Deskripsi |
|------|-----------|
| `frontend/src/stores/compare.js` | Pinia store - manage compare list, persist localStorage |
| `backend/swagger.json` | OpenAPI 3.0.3 spec - 28 endpoints documented |
| `backend/src/server.js` | Register `@fastify/swagger` + `@fastify/swagger-ui` |

## 🔧 Perubahan Service Layer

- `pendaftaran.create(formData)` - multipart form upload
- `admin.*` - 9 endpoints (stats, users, pesantren CRUD, pendaftaran management, export)
- `pemilik.*` - 3 endpoints (pesantren CRUD own)
- `wilayah.*` - 4 endpoints (provinces, regencies, details)
- `auth.updateMe(data)` - update profile
