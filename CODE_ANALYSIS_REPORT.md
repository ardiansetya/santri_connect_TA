## 🔴 CRITICAL ISSUES <a name="critical-issues"></a>

### 1. **JWT Secret Mismatch** ⚠️ SECURITY RISK
**Severity:** HIGH  
**Files:**
- `backend/src/server.js` (line ~28): `secret: process.env.JWT_SECRET || 'santri_connect_secret_key'`
- `backend/gen-token.js` (line 5): `secret: 'santo_connect_secret_key_2024'`
- `backend/gen-token-pendaftar.js`: Different secret
- `backend/gen-token-pendaftar2.js`: Different secret
- `backend/test/helper.js`: `secret: 'test_secret_key_for_testing'`

**Problem:** Token generation scripts use different secrets than the server. Tokens generated by these scripts will **fail authentication** in development.

**Impact:** 
- Tokens generated for testing won't work
- "santo" vs "santri" typo indicates copy-paste error

**Fix:** 
```javascript
// All files should use:
secret: process.env.JWT_SECRET || 'santri_connect_secret_key'
```

### 5. **Incomplete Data Loading** ⚠️ BROKEN FEATURE
**Severity:** MEDIUM  
**Files:**
- `frontend/src/components/dashboard/PendaftarDashboard.vue`
- `frontend/src/components/dashboard/PemilikDashboard.vue`

**Problem:** Components set `loading.value = false` in `onMounted` but **never fetch data**:

```javascript
onMounted(async () => {
  loading.value = false  // Just disables loading, no data fetch!
})
```

**Impact:** Dashboards show empty state even when data exists.

**Fix:** Implement proper data fetching:
```javascript
onMounted(async () => {
  try {
    const response = await someApi.getData()
    data.value = response.data
  } catch (error) {
    console.error('Failed to load data:', error)
  } finally {
    loading.value = false
  }
})
```

### 10. **Missing Error Handling in Async Routes**
**Severity:** MEDIUM  
**Files:** All controller files

**Problem:** Controllers don't use try-catch consistently. Fastify 5 requires proper error handling.

**Example of bad pattern:**
```javascript
async function getList(request, reply) {
  const result = await PesantrenService.getList(request.query)
  reply.send(result)  // No error handling!
}
```

**Recommendation:** 
```javascript
async function getList(request, reply) {
  try {
    const result = await PesantrenService.getList(request.query)
    return reply.send(result)
  } catch (error) {
    request.log.error(error)
    return reply.status(500).send({
      error: 'Internal server error',
      message: process.env.NODE_ENV === 'development' ? error.message : undefined
    })
  }
}
```

Or use Fastify's error handler globally in `server.js`.

## 🏗️ ARCHITECTURE & BEST PRACTICES <a name="best-practices"></a>

### Backend Architecture (Fastify 5)

#### Current Pattern: MVC + Service Layer
```
Request → Route → Controller → Service → Model → Database
```

#### ✅ Good Practices Found:
1. Separation of concerns (controllers don't touch database directly)
2. Connection pooling via `mysql2`
3. JWT-based authentication
4. Middleware for validation (`validate-wilayah.js`)
5. Swagger documentation setup
6. Jest testing with Supertest

#### ❌ Issues vs Fastify Best Practices:

**1. Monolithic Routes File**
- **File:** `backend/src/routes/index.js` contains ALL routes
- **Best Practice:** Use `@fastify/autoload` for automatic route registration
- **Fix:**
```bash
npm install @fastify/autoload
```
```javascript
// server.js
await fastify.register(require('@fastify/autoload'), {
  dir: path.join(__dirname, 'routes'),
  options: { prefix: '/api' }
})
```

**2. No Request Validation Schemas**
- **Best Practice:** Fastify excels at JSON Schema validation
- **Current:** Manual validation in controllers
- **Fix:** Define schemas per route:
```javascript
{
  schema: {
    body: {
      type: 'object',
      required: ['email', 'password'],
      properties: {
        email: { type: 'string', format: 'email' },
        password: { type: 'string', minLength: 8 }
      }
    }
  }
}
```

**3. No Plugin Architecture**
- **Best Practice:** Fastify's plugin encapsulation for isolation
- **Current:** Everything registered globally
- **Fix:** Group related routes as plugins:
```javascript
// plugins/auth.js
const authPlugin = require('fastify-plugin')(async function (fastify) {
  fastify.register(require('../routes/auth-routes'))
})
module.exports = authPlugin
```

**4. Missing Health Check Endpoint**
```javascript
// Add to server.js
fastify.get('/health', async () => {
  try {
    await fastify.mysql.query('SELECT 1')
    return { status: 'ok', database: 'connected' }
  } catch (error) {
    return reply.status(503).send({ status: 'error', database: 'disconnected' })
  }
})
```

**5. No Rate Limiting**
```bash
npm install @fastify/rate-limit
```
```javascript
await fastify.register(require('@fastify/rate-limit'), {
  max: 100,
  timeWindow: '1 minute'
})
```

### Frontend Architecture (Vue 3)

#### ✅ Good Practices Found:
1. Composition API with `<script setup>`
2. Pinia for state management
3. Axios interceptors for auth headers
4. Vue Router for navigation
5. Vee-Validate + Yup for form validation

#### ❌ Issues vs Vue 3 Best Practices:

**1. No Composables for Reusable Logic**
- **Problem:** Dashboard components duplicate status badge logic
- **Best Practice:** Extract to composables:
```javascript
// src/composables/useStatusBadge.js
export function useStatusBadge() {
  const statusBadge = (status) => {
    const map = {
      pending: 'badge-pending',
      diterima: 'badge-approved'
    }
    return map[status] || 'badge-default'
  }
  return { statusBadge }
}
```

**2. Direct API Calls in Components**
- **Current:** Components import and call API services directly
- **Best Practice:** Use composable pattern or store actions:
```javascript
// Bad
import { pesantren } from '../../services'
const data = await pesantren.getList()

// Good
import { usePesantrenStore } from '@/stores/pesantren'
const store = usePesantrenStore()
await store.fetchPesantren()
```

**3. No Error Boundaries**
- Add global error handler in `main.js`:
```javascript
app.config.errorHandler = (err, instance, info) => {
  console.error('Vue Error:', err)
  // Send to error tracking service
}
```

**4. Missing Loading States Management**
- Use VueUse's `useAsyncState` or create custom composable:
```javascript
import { useAsyncState } from '@vueuse/core'

const { state, isLoading, execute } = useAsyncState(
  () => api.getData(),
  null
)
```

**5. No Component Documentation**
- Add JSDoc to complex components:
```javascript
/**
 * Dashboard for admin users showing statistics
 * @component
 * @example
 * <AdminDashboard />
 */
```

## 📚 REFERENCES & OFFICIAL DOCUMENTATION <a name="references"></a>

### Backend (Fastify 5)

Based on official Fastify documentation via Context7:

1. **Route Organization Best Practice:**
   - Use route prefixing: `fastify.register(routes, { prefix: '/v1' })`
   - Modular route files with schemas for validation
   - Source: [Fastify Routes Documentation](https://github.com/fastify/fastify/blob/main/docs/Reference/Routes.md)

2. **Schema Validation:**
   - Fastify excels at JSON Schema validation for request/response
   - Define schemas per route for automatic validation
   - Source: [Context7 Fastify Docs](/fastify/fastify)

3. **Plugin Architecture:**
   - Use `fastify-plugin` for encapsulation
   - Each feature module should be a plugin
   - Source: [Fastify Plugin Pattern](https://fastify.dev/docs/latest/Reference/Plugins/)

4. **JWT Best Practices:**
   - Never hardcode secrets
   - Use environment variables exclusively
   - Source: [Fastify JWT Docs](/fastify/fastify-jwt)

### Frontend (Vue 3)

Based on official Vue.js documentation via Context7:

1. **File Naming Convention (Official Style Guide):**
   - **Component files:** `PascalCase.vue` (e.g., `MyComponent.vue`)
   - **Base components:** Prefix with `Base` (e.g., `BaseButton.vue`)
   - **Tightly coupled child components:** Prefix with parent name (e.g., `TodoItem.vue`, `TodoList.vue`)
   - **Source:** [Vue Style Guide Discussion](https://github.com/orgs/vuejs/discussions/14241)

2. **Composable Naming Pattern:**
   - **Function names:** Must start with `use` (e.g., `useAuth()`, `usePesantren()`)
   - **File names:** Match function name (e.g., `useAuth.js`, `usePesantren.js`)
   - **Must return:** Plain object containing refs/reactive state
   - **Source:** [Vue Composables Best Practices](https://dev.to/jacobandrewsky/good-practices-and-design-patterns-for-vue-composables-24lk)

3. **Composition API Pattern:**
   - Use `<script setup>` syntax (already done ✅)
   - Use `ref()` for primitives, `reactive()` for objects
   - Use `computed()` for derived state
   - Extract reusable logic to composables
